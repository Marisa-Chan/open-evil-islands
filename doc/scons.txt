   * У SCons отличная документация, есть объёмное руководство пользователя,
     есть man-страница. Всё это доступно здесь:
     http://www.scons.org/documentation.php

   * Справка по опциям SCons:
     scons -H

   * Справка по опциям сборки (эту справку, как и сами опции, задаём мы):
     scons -h

   * Доступные цели SCons (предполагаю, что с терминологией все знакомы).
     Собрать всё (аналог make all):
     scons или scons all

     Собрать только библиотеку и game.exe:
     scons game

     Собрать starter.exe (пока недоступно):
     scons starter

     Собрать все спайки (напомню, спайки зависят от игровой библиотеки,
     так что сборка для последней запустится также в случае необходимости):
     scons spikes

     Собрать только один или несколько конкретных спайков:
     scons mprviewer
     scons mprviewer texviewer ...

     Собрать внутриигровые утилиты (пока недоступно):
     scons tools

     Собрать конкретную утилиту (пока недоступно):
     scons resfileviewer

   * Доступные опции сборки.
     RELEASE=<true,yes|false,no> - Собрать в релизе (по умолчанию включен):
     scons RELEASE=no texviewer

     HOST=<name> - Собрать, используя указанную конфигурацию (это может
     быть другой компилятор, кросс-компилятор или ещё что-то; по умолчанию
     сборка запускается для той же платформы, из-под которой запущен SCons):
     scons HOST=i386-linux-mingw spikes

     LOGGING_LEVEL=<level> - Установить уровень вывода лога сборки (не игры!):
     scons LOGGING_LEVEL=debug all

     Остальное несущественно или интуитивно понятно.

   * Дизайн опций сборки.
     Финальные опции формируются проходя через несколько ступеней.
     Это сделано для удобства.

     Наивысшим приоритетом обладают опции, введёные вручную, например:
     scons RELEASE=no tools

     Чуть ниже приоритет у опций, указанных в файле cursedearth_local.cfg
     (этот файл нужно создать самим по примеру cursedearth_local.cfg.example,
     под Git не класть, это только ваши локальные опции):
     [CE]
     RELEASE = no

     Ещё ниже приоритет у опций, указанных в файле cursedearth.cfg
     (этот файл, в отличие от cursedearth_local.cfg, под Git,
     в нём находятся опции по умолчанию, его лучше не редактировать):
     [CE]
     RELEASE = yes

     И, наконец, самый низкий приоритет у опций по умолчанию,
     зашитых в скрипты сборки.

   * Удалить все созданные SCons'ом файлы (аналог make clean):
     scons -c
     scons texviewer -c
