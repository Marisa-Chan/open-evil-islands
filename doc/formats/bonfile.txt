   * Введение.
     Каждая часть модели, представляемая одноимённым файлом fig, изначально
     находится в начале системы координат. Формат bon определяет позу привязки
     (bind pose) для этой части. Например, поза привязки для головы -
     это верх туловища.

   * Описание.
     Формат очень прост - нет сигнатуры, нет полей с количеством элементов.
     Количество элементов в файле bon можно узнать из соответствующего
     файла fig. Подробности в figfile.

     В файле уложен поток чисел float. Их количество - n x 3, где
     3 - компоненты x, y, z;
     n - количество вариантов для каждой компоненты.

     Количество вариантов нужно для расчёта позы для конкретной комплекции.
     Для формата fig8 количество вариантов 8, для fig6 - 6.
     Подробности в figfile.

   * Пример.
     Модель unmotr (тролль), часть - hd (голова).

         x         y       z
     0.000000 -0.361341 1.110468   1
     0.000000 -0.361341 1.110468   2
     0.000000 -0.361341 1.110468   3
     0.000000 -0.361341 1.110468   4
     0.000001 -0.735542 2.260456   5
     0.000001 -0.735542 2.260456   6
     0.000001 -0.735542 2.260456   7
     0.000001 -0.735542 2.260456   8
